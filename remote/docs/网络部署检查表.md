# 网络部署检查表

按下面顺序检查，可以快速定位问题发生在当前电脑、Linux 工控机、ARTIQ core device，还是同在 `hku_ultracold` 局域网中的 Moku:Go。

## 1. 当前电脑到 Linux 工控机

在当前电脑 PowerShell 中：

```powershell
ping 工控机IP
ssh 用户名@工控机IP
```

如果 `ping` 不通：

- 检查网线、交换机、网口灯。
- 检查两台电脑是否在同一网段，或路由是否正确。
- 检查 Windows 防火墙和 Linux 防火墙。

如果 `ping` 通但 `ssh` 不通：

- 检查工控机是否启动 SSH 服务：`sudo systemctl status ssh` 或 `sudo systemctl status sshd`。
- 检查端口是否是 22，或者在 `config.local.json` 中修改 `ssh_port`。
- 检查用户名、密码或 SSH 密钥。

## 2. Linux 工控机到 ARTIQ core device

登录工控机后：

```bash
ping ARTIQ_core_device_IP
```

检查 `device_db.py`：

```bash
grep -n "core_addr" device_db.py
```

`core_addr` 应该是 ARTIQ core device 的 IP，而不是当前电脑的 IP。

## 3. 工控机本地 ARTIQ

在工控机上：

```bash
which artiq_run
artiq_run --version
artiq_run examples/core_reset.py
```

如果本机 `artiq_run` 都无法运行，先不要调远程脚本，需要先修复工控机的 ARTIQ 安装或硬件连接。

## 4. 当前电脑调用远端 ARTIQ

在本目录：

```powershell
python remote_artiq.py check --config config.local.json --deep
python remote_artiq.py run --config config.local.json examples\core_reset.py
```

如果 `--deep` 找不到 `artiq_run`，通常是 SSH 非交互环境没有加载 ARTIQ。请设置 `remote_env_setup`。

## 5. artiq_master 模式端口

如果使用 `artiq_master/artiq_client`：

```powershell
python remote_artiq.py check --config config.local.json
```

默认需要能访问：

- control: `工控机IP:3251`
- notify: `工控机IP:3250`

如果端口不通，检查：

- 工控机上是否已经启动 `artiq_master --bind 0.0.0.0`。
- Linux 防火墙是否放行 3250 和 3251。
- `config.local.json` 中 `artiq_master_host` 是否正确。

## 6. Moku:Go 同网段检查

Moku:Go 不属于 ARTIQ 实时链路，但它和当前电脑、Linux 工控机、ARTIQ 设备都在 `hku_ultracold` 局域网中。

在当前电脑上：

```powershell
cd /d "Z:\Wang Junjie\Artiq_Project\observer\moku"
python moku_tools.py check --config moku_config.local.json
```

如果只想发现设备：

```powershell
mokucli list
```

如果当前电脑发现不了 Moku:Go：

- 检查 Moku:Go 是否接入 `hku_ultracold`。
- 检查当前电脑是否连接同一个路由器/同一网段。
- 检查 Windows 防火墙是否阻止局域网发现。
- 检查 Moku:Go 是否被其他电脑或 Moku App 占用。
